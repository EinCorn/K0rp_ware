# K0rp_OS UI assets V3 — source and validation contract

## Scope

Task 022A(2.1), merged in PR #39, establishes the trustworthy source boundary. Task 022A(2.2) materializes only the maintained pilot allowlist and uses it for the Audit 00-A and Formuláře window chrome.

```text
design/ui-source/k0rp-os-ui-assets-v3/       canonical raw source snapshot
design/ui-runtime/k0rp-v3/inventory.json     generated normalized inventory
design/ui-runtime/k0rp-v3/runtime-allowlist.json maintained Task 022A(2.2) boundary
src/assets/ui/korp-v3/                       generated byte-identical @2x pilot subset
src/ui/korpUiAssetCatalog.js                 generated semantic runtime catalog
```

Commands:

```text
npm run build:korp-ui-assets     regenerate only the normalized inventory
npm run build:korp-ui-runtime    regenerate the bounded runtime subset and catalog
npm run validate:korp-ui-assets  validate source, allowlist, subset and generated drift
```

All commands are offline and use only Node built-ins. Neither generator rewrites the raw snapshot, its checksums or source metadata.

## Validation and readiness model

`npm run validate:korp-ui-assets` fails closed for:

- malformed required JSON or manifest structure;
- duplicate semantic IDs, duplicate format paths or duplicate runtime selections;
- missing production assets, unsafe/path-escaping paths or manifest path mismatches;
- native, `@2x` or WebP dimensions that disagree with headers;
- malformed/out-of-bounds content rectangles or cap insets;
- an incomplete materialized nine-slice family;
- invalid atlas/window-metric/QA references;
- unknown, reference-only or icon-like runtime allowlist selections;
- checksum mismatches for files that exist;
- generated inventory drift;
- a deterministic runtime-source reference to the raw V3 root;
- an unmanaged byte-identical raw source copy outside `src/assets/ui/korp-v3`;
- a missing, extra, dimension-drifted or byte-drifted file inside the sanctioned runtime subset;
- generated runtime catalog drift.

Every normalized asset is classified as `core-now`, `derived-piece`, `later-state`, `reference-only` or `flavor-later`. The inventory stores relative source paths, intrinsic/header dimensions, byte sizes, SHA-256 values, category/role, production status, content/nine-slice metadata and pilot selection state. It contains no absolute Windows paths or timestamps.

### Windows preflight diagnosis

On the Task 022A(2.2) branch, `npm run validate:korp-ui-assets` reproduced the reported local message with exit code `0`. The validator itself passed and all 15 original tests passed. The confusing part was output routing: four stable warnings used `console.warn`, so PowerShell treated the first stderr line as `NativeCommandError` and displayed `CategoryInfo` / `FullyQualifiedErrorId` metadata after the successful TAP output.

This was warning-only, not a broken validation or build. The validator now prints `PASS WITH 4 KNOWN SOURCE WARNINGS` and four explicit `WARN [...]` records to stdout. Actual malformed or missing production fixtures still print `FAIL` to stderr and exit non-zero. Regression coverage also verifies LF/CRLF-equivalent checksum parsing and that warning-only validation never mutates the raw snapshot.

## Task 022A(2.2) runtime allowlist

The maintained pilot boundary contains exactly 45 unique, production, text-free semantic IDs:

| Pilot group | Count | Boundary |
|---|---:|---|
| Audit window | 11 | `window.audit.frame`, `window.audit.content`, 9 frame pieces |
| Formuláře folder window | 12 | frame, content/list surfaces, 9 frame pieces |
| Titlebar materials | 2 | active and inactive blanks |
| Window controls | 8 | close/minimize × disabled/hover/normal/pressed |
| Audit controls | 11 | 4 button, 4 checkbox and 3 radio states |
| Folder row | 1 | blank resizable row |

Task 022A(2.1) copied zero assets. Task 022A(2.2) changes `copyAssets` to `true` and deterministically materializes exactly these 45 `@2x` PNGs. The generated catalog is ordered by semantic ID and records URL, intrinsic/native dimensions, scale, group, family, role, state, semantic, derivation and geometry metadata. Unknown, reference-only, baked/template, composite or icon-like selections fail closed.

The boundaries remain separate:

```text
raw source snapshot
!= normalized inventory
!= maintained allowlist
!= generated runtime subset
!= semantic runtime catalog
```

`npm run build:korp-ui-runtime` writes `src/assets/ui/korp-v3/` and `src/ui/korpUiAssetCatalog.js`; `--check` verifies exact filenames, bytes, PNG dimensions, catalog order and the absence of extras. The source validator permits byte-identical copies only in this exact generated root and continues to reject raw copies elsewhere.

The fixed `composite_blank` files remain source alternatives to `content → runtime content → frame` composition and are deliberately excluded from the pilot boundary. The independent baked `document.audit_00a.template` is also excluded: reference/example screens and document templates are not player-facing runtime copy. Existing React data remains authoritative for Audit text and Formuláře status text.

Canonical icons continue to come only from `design/icon-source/k0rp-icons-v2` and its generated runtime subset. ClickAudit and Fidget retain their existing asset-backed module surfaces. Compact/standard modules, memo/modal/toast/lock families, bars, taskbar, extra controls, status language, textures and flavor surfaces remain reserved for Tasks 022A(2.3–2.5) or later.

Only Audit 00-A and Formuláře use the V3 primitive in this pilot. Audit 10-A, Audit 16-C, Doručené, Denní výpis, ClickAudit, Fidget, the top rail and taskbar retain their current renderers. Task 022A(2.3) remains next for Audit 10-A, Audit 16-C, Doručené and Denní výpis; Task 023 gameplay stays deferred until the bounded visual pass is complete.
